This reference details the execution flow and responsibility of every job in the Forge.js ecosystem.

## Pipeline Lifecycle

Forge.js workflows are designed to be modular. Below is the logical progression from code change to production release.

<Steps>
1.  **Quality Assurance**  
    Checks are performed on every push using `quality.yml` and `testing.yml`.
2.  **Build & Artifacts**  
    Once quality passes, `build.yml` compiles the source code into a redistributable format.
3.  **Versioning**  
    `version.yml` calculates the next semantic version and prepares release notes.
4.  **Delivery**  
    Finally, `library.yml` (NPM) or `debian.yml` (Linux) publishes the artifacts to registries.
</Steps>

---

## default.yml
*The foundation. These jobs provide the environment used by all other pipelines.*

<Steps >
1.  <Card title="setup" icon="rocket">
      **Environment Setup**  
      Prepares Node.js, caches dependencies, performs a clean install (`npm ci`), and runs a high-level security audit.
      <Badge text="Standalone" variant="note" />
    </Card>
2.  <Card title="auto-merge" icon="git-merge">
      **Dependabot Automation**  
      Automatically merges Dependabot pull requests for minor and patch updates to keep dependencies fresh.
      <Badge text="Standalone" variant="note" />
    </Card>
</Steps >

## quality.yml
*Ensures code standards are maintained before any further processing.*

<Steps>
1.  **Prepare Environment**  
    <LinkCard title="Calls ➔ default.yml" href="#defaultyml" description="Triggers the setup job to prepare Node.js and dependencies." />
2.  <Card title="check" icon="approve-check-circle">
      **Static Analysis**  
      Executes Biome linting and TypeScript type checking in parallel.
      <Badge text="Needs: setup" variant="tip" />
    </Card>
</Steps>

## testing.yml
*The gatekeeper for logic correctness.*

<Steps>
1.  **Verify Quality**  
    <LinkCard title="Calls ➔ quality.yml" href="#qualityyml" description="Ensures linting and types are perfect before starting tests." />
2.  <Card title="test" icon="beaker">
      **Unit Tests**  
      Runs the full Vitest suite and uploads code coverage reports.
      <Badge text="Needs: quality" variant="tip" />
    </Card>
</Steps>

## build.yml
*Compiles the source code into production-ready assets.*

<Steps>
1.  **Prepare Environment**  
    <LinkCard title="Calls ➔ default.yml" href="#defaultyml" description="Triggers the setup job." />
2.  <Card title="build" icon="puzzle">
      **Compilation**  
      Bundles the project using Tsup and uploads the resulting `dist` directory.
      <Badge text="Needs: setup" variant="tip" />
    </Card>
</Steps>

## version.yml
*The brain of the release process.*

<Steps>
1.  **Verify Build**  
    <LinkCard title="Calls ➔ build.yml" href="#buildyml" description="Ensures the project compiles before attempting a release." />
2.  <Card title="version" icon="tag">
      **Semantic Release**  
      Analyzes commits, determines versioning, and creates a GitHub Release.
      <Badge text="Needs: build" variant="tip" />
    </Card>
</Steps>

## library.yml
*NPM publishing orchestration.*

<Steps>
1.  **Prepare Release**  
    <LinkCard title="Calls ➔ version.yml" href="#versionyml" description="Triggers the build and versioning logic." />
2.  <Card title="document" icon="document">
      **API Documentation**  
      Generates API documentation via TypeDoc and deploys it to GitHub Pages.
      <Badge text="Needs: version" variant="tip" />
    </Card>
3.  <Card title="publish" icon="upload">
      **Registry Publish**  
      Publishes the package to NPM or GitHub Packages with OIDC-based provenance.
      <Badge text="Needs: document" variant="tip" />
    </Card>
</Steps>

## debian.yml
*Linux packaging orchestration.*

<Steps>
1.  **Prepare Release**  
    <LinkCard title="Calls ➔ version.yml" href="#versionyml" description="Triggers the build and versioning logic." />
2.  <Card title="build" icon="linux">
      **Debian Packaging**  
      Converts build artifacts into a `.deb` installer using Snodeb and uploads the package.
      <Badge text="Needs: version" variant="tip" />
    </Card>
</Steps>
