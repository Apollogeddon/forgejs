This directory contains reusable GitHub Actions workflows designed for Node.js projects.
These provide a modular CI/CD pipeline covering quality checks, testing, building, versioning, documentation, publishing, and Debian package creation.

## Setup Guide

To integrate these workflows, reference them using the `uses` keyword in the project's GitHub Actions workflow files.

### Usage Examples

<Tabs>
  <TabItem label="Basic Testing" icon="seti:vite">
    For projects requiring only test execution on push.

    ```yaml
    jobs:
      testing:
        uses: apollogeddon/forgejs/.github/workflows/testing.yml@main
    ```
  </TabItem>
  <TabItem label="Full Versioning" icon="rocket">
    For applications requiring versioning and release management.

    ```yaml
    jobs:
      version:
        uses: apollogeddon/forgejs/.github/workflows/version.yml@main
        secrets:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
    ```
  </TabItem>
  <TabItem label="Application CI" icon="laptop">
    For Backends or Websites requiring Testing and Versioning.

    ```yaml
    jobs:
      ci:
        uses: apollogeddon/forgejs/.github/workflows/application.yml@main
        with:
          node_version: '22'
        secrets: inherit
    ```
  </TabItem>
  <TabItem label="NPM Publishing" icon="seti:npm">
    For libraries publishing to an npm registry.

    ```yaml
    jobs:
      publish:
        uses: apollogeddon/forgejs/.github/workflows/library.yml@main
        with:
          artifact_name: 'my-lib-dist'
        secrets:
          NPM_TOKEN: ${{ secrets.NPM_TOKEN }}
    ```
  </TabItem>
</Tabs>

### Common Inputs

- **`working_directory`** (string, default: `'.'`): Path to the directory containing `package.json`.
- **`node_version`** (string, default: `'22'`): The version of Node.js to use.
- **`auto_patch`** (boolean, default: `false`): If enabled, automatically remediates vulnerabilities found by OSV-Scanner and commits them back to the repository.
- **`artifact_name`** (string, default: `'dist'`): The name of the artifact to upload/download between build and publish/package steps. _(Available in `build`, `library`, and `debian` workflows)_
- **`enable_secrets`** (boolean, default: `true`): Controls whether secrets scanning (Gitleaks) is enabled in workflows that include it.

### Common Secrets

- **`NPM_TOKEN`**: Required for publishing to private npm registries via `library.yml`.
- **`GITHUB_TOKEN`**: Automatically provided. For security scanning workflows (OSV-Scanner) to upload results to GitHub Code Scanning, the `GITHUB_TOKEN` requires `security-events: write` and `actions: read` permissions. It is also used for `release-please`, GitHub Packages, and Dependabot automation.

...

permissions:
  contents: write # Required for release-please (tagging, changelog)
  packages: write # Required for publishing to GitHub Packages
  id-token: write # Required for provenance generation
  security-events: write # Required for uploading Sarif reports (OSV-Scanner)
```
