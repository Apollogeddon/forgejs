This reference details the execution flow and responsibility of each job within the workflow ecosystem.

## Pipeline Lifecycle

The workflows follow a modular design. The logical progression from code change to production release is as follows:

<Steps>
1.  **Quality Assurance**  
    The pipeline runs quality checks on every push using `quality.yml` and `testing.yml`.
2.  **Build & Artifacts**  
    Upon successful quality checks, `build.yml` compiles the source code into a redistributable format.
3.  **Versioning**  
    `version.yml` calculates the next semantic version and prepares release notes.
4.  **Delivery**  
    `library.yml` (NPM) or `debian.yml` (Linux) publishes the artifacts to registries.
</Steps>

## Default.yml
*The foundation. These jobs provide the environment used by all other pipelines.*

<Steps >
1.  <Card title="setup" icon="setting">
      **Environment Setup**  
      Prepares Node.js, caches dependencies, performs a clean install (`npm ci`), and runs a high-level security audit.
      <Badge text="Standalone" variant="note" />
    </Card>
2.  <Card title="auto-merge" icon="seti:git">
      **Dependabot Automation**  
      Automatically merges Dependabot pull requests for minor and patch updates to maintain dependency freshness.
      <Badge text="Standalone" variant="note" />
    </Card>
</Steps >

## Quality.yml
*Ensures code standards are maintained and security is audited before further processing.*

<Steps>
1.  **Prepare Environment**  
    <LinkCard title="Calls ➔ default.yml" href="#defaultyml" description="Triggers the setup job to prepare Node.js and dependencies." />
2.  <Card title="check" icon="approve-check-circle">
      **Security & Static Analysis**  
      Executes Biome linting, OSV-Scanner vulnerability audit, and TypeScript type checking.
      <Badge text="Needs: setup" variant="tip" />
    </Card>
</Steps>

## Testing.yml
*Validates logic correctness.*

<Steps>
1.  **Verify Quality**  
    <LinkCard title="Calls ➔ quality.yml" href="#qualityyml" description="Ensures linting and types are correct before starting tests." />
2.  <Card title="test" icon="seti:vite">
      **Unit Tests**  
      Runs the full Vitest suite and uploads code coverage reports.
      <Badge text="Needs: quality" variant="tip" />
    </Card>
</Steps>

## Build.yml
*Compiles source code into production-ready assets.*

<Steps>
1.  **Prepare Environment**  
    <LinkCard title="Calls ➔ default.yml" href="#defaultyml" description="Triggers the setup job." />
2.  <Card title="build" icon="node">
      **Compilation**  
      Bundles the project using Tsup and uploads the resulting `dist` directory.
      <Badge text="Needs: setup" variant="tip" />
    </Card>
</Steps>

## Application.yml
*Orchestrates the pipeline for Applications (Backends/Websites).*

<Steps>
1.  **Testing**  
    <LinkCard title="Calls ➔ testing.yml" href="#testingyml" description="Runs quality checks and unit tests." />
2.  **Versioning**  
    <LinkCard title="Calls ➔ version.yml" href="#versionyml" description="Calculates version, releases, and updates the artifact." />
</Steps>

## Version.yml
*Manages the release process.*

<Steps>
1.  <Card title="release-please" icon="rocket">
      **Release Please**  
      Analyses commits, determines versioning, and creates a Release Pull Request or creates a GitHub Release.
      <Badge text="On Main" variant="tip" />
    </Card>
2.  **Verify Build**  
    <LinkCard title="Calls ➔ build.yml" href="#buildyml" description="Ensures the project compiles after a release is triggered." />
</Steps>

## Library.yml
*Orchestrates NPM publishing.*

<Steps>
1.  **Prepare Release**  
    <LinkCard title="Calls ➔ version.yml" href="#versionyml" description="Triggers the build and versioning logic." />
2.  <Card title="document" icon="document">
      **API Documentation**  
      Generates API documentation via TypeDoc and deploys it to GitHub Pages.
      <Badge text="Needs: version" variant="tip" />
    </Card>
3.  <Card title="publish" icon="seti:npm">
      **Registry Publish**  
      Publishes the package to NPM or GitHub Packages with OIDC-based provenance.
      <Badge text="Needs: document" variant="tip" />
    </Card>
</Steps>

## Debian.yml
*Orchestrates Linux packaging.*

<Steps>
1.  **Prepare Release**  
    <LinkCard title="Calls ➔ version.yml" href="#versionyml" description="Triggers the build and versioning logic." />
2.  <Card title="build" icon="linux">
      **Debian Packaging**  
      Converts build artifacts into a `.deb` installer using Snodeb and uploads the package.
      <Badge text="Needs: version" variant="tip" />
    </Card>
</Steps>
