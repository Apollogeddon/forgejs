Forge.js provides **tooling configurations** and **GitHub Actions workflows** for TypeScript and Node.js projects.

## Setup Guide

<Steps>

1.  **Installation**

    Install the package as a dev dependency:

    ```bash
    npm install --save-dev @apollogeddon/forgejs
    ```

2.  **Initialisation**

    Use the included CLI to bootstrap the project. You can choose a project mode and toggle specific features.

    **Quick Start**
    ```bash
    # Initialize a Backend/Service (Default)
    npx @apollogeddon/forgejs init

    # Initialize a Library (includes TypeDoc)
    npx @apollogeddon/forgejs init --library

    # Initialize a Website (Vite/Astro)
    npx @apollogeddon/forgejs init --website
    ```

    **CLI Options**
    Customise the features you need using flags:

    | Flag | Description | Included Features |
    | :--- | :--- | :--- |
    | `--backend` | Setup for Backend/Service (Default) | `tsup`, `distroless` docker support, `application.yml` |
    | `--library` | Setup for TypeScript Library | `tsup`, `typedoc`, `library.yml` |
    | `--website` | Setup for Frontend Website | `vite`, `nginx` docker support, `application.yml` |
    | `--debian` | Enable Debian packaging | `snodeb.config.cjs`, `debian.yml` |
    | `--docker` | Enable Dockerfile generation | `Dockerfile`, `.dockerignore` |
    | `--version` | Semantic Versioning (Default) | `commitlint.config.ts` |
    | `--testing` | Testing framework (Default) | `vitest.config.ts`, `tests/` |
    | `--linting` | Linting & Formatting (Default) | `biome.json`, `lefthook.yml` |
    | `--typedoc` | API Documentation (Default for Library) | `astro` configs, `src/content/docs` |
    | `--dry-run` | Simulate execution | Logs changes without writing files |
    | `--force` | Overwrite existing files | Enforces standard configs |
    | `--help` | Show help | Displays usage information |

    **Example: Backend with Docker & Debian**
    ```bash
    npx @apollogeddon/forgejs init --backend --docker --debian
    ```

    The command performs the following actions:
    *   Creates configuration files (e.g., `biome.json`, `vitest.config.ts`, `lefthook.yml`) based on selection.
    *   Sets `type: "module"` in `package.json`.
    *   Injects relevant scripts into `package.json`.

    **Generated Scripts**
    The following scripts are added depending on the active features:

    | Script | Command | Description |
    | :--- | :--- | :--- |
    | `watch` | `tsx watch src/index.ts` | Watches the project entry point (Backend). |
    | `start` | `node dist/index.js` | Runs the built artifact (Backend). |
    | `dev` | `vite` | Starts dev server (Website). |
    | `lint` | `biome check --fix` | Runs Biome to lint and format code. |
    | `type` | `tsc --noEmit` | Runs TypeScript type checking. |
    | `build` | `tsup` / `vite build` | Builds the project. |
    | `test` | `vitest run` | Runs unit tests. |
    | `docs:init` | `astro sync` | Initialises Astro types (Library/TypeDoc). |
    | `docs:build` | `astro build` | Builds documentation site (Library/TypeDoc). |
    | `docs:watch` | `astro dev` | Starts documentation dev server (Library/TypeDoc). |
    | `build:deb` | `snodeb` | Builds Debian package (Debian). |
    | `docker:build`| `docker build ...` | Builds Docker image (Docker). |
    | `publint` | `publint` | Checks package compatibility (Library). |
    | `prepare` | `lefthook install` | Installs Git hooks. |

3.  **Advanced: Overwriting Files**

    To overwrite existing configuration files with the library defaults, use the `--force` flag. You can verify what will happen first using `--dry-run`.

    ```bash
    npx @apollogeddon/forgejs init --dry-run
    npx @apollogeddon/forgejs init --force
    ```

</Steps>

## Project Structure

A standard project utilising this library adheres to the following structure:

<FileTree>
  - .github/
    - workflows/
      - ci.yml
  - src/
    - content/
      - docs/
        - index.mdx
    - index.ts
  - tests/
    - index.test.ts
  - .editorconfig
  - astro.config.mjs
  - biome.json
  - commitlint.config.ts
  - lefthook.yml
  - package.json
  - snodeb.config.ts
  - tsconfig.doc.json
  - tsconfig.json
  - tsup.config.ts
  - vitest.config.ts
</FileTree>
