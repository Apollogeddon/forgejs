`@apollogeddon/forgejs` provides **standardised tooling configurations** and **reusable GitHub Actions workflows** designed to streamline DevOps for TypeScript and Node.js projects.

## Setup Guide

<Steps>

1.  **Installation**

    Install the package as a dev dependency:

    ```bash
    npm install --save-dev @apollogeddon/forgejs
    ```

2.  **Initialisation**

    To quickly set up your project with the recommended configurations and scripts, use the included CLI.

    ```bash
    npx @apollogeddon/forgejs init
    ```

    This command will:
    *   Create configuration files (e.g., `biome.json`, `vitest.config.ts`, `lefthook.yml`) if they don't exist.
    *   Ensure `type: "module"` is set in your `package.json`.
    *   Add recommended scripts (`lint`, `test`, `build`, etc.) to your `package.json`.

3.  **Advanced: Overwriting Files**

    If you need to overwrite existing configuration files with the defaults from this library, use the `--force` flag:

    ```bash
    npx @apollogeddon/forgejs init --force
    ```

</Steps>

## Recommended Project Structure

Here is how a standard project using this library should look:

<FileTree>
- .github/
  - workflows/
    - index.yml          # Imports workflows from this library
- src/
  - index.ts               # Source code
- tests/
  - index.test.ts          # Unit Tests
- .editorconfig              # Editor style settings
- .releaserc.json            # Semantic Release config
- biome.json                 # Linter/Formatter config
- commitlint.config.ts       # Commit linter config
- lefthook.yml               # Git hooks config
- package.json               # Dependencies & scripts
- snodeb.config.ts           # Debian package config (optional)
- tsconfig.json              # TypeScript config
- tsup.config.ts             # Build/Bundler config
- typedoc.json               # Documentation generator config
- vitest.config.ts           # Test runner config
</FileTree>
