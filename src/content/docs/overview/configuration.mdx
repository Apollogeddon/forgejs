The project exports configurations for standard tooling. These can be extended to inherit best practices while allowing for project-specific overrides.

## Quality & Testing

<Tabs>
  <TabItem label="Biome" icon="biome">
    **Linting & Formatting**

    Extend the shared Biome configuration in `biome.json`:

    ```json
    {
      "$schema": "node_modules/@biomejs/biome/configuration_schema.json",
      "extends": ["node_modules/@apollogeddon/forgejs/biome.json"]
    }
    ```
  </TabItem>

  <TabItem label="Vitest" icon="seti:vite">
    **Testing**

    Import and merge the base configuration in `vitest.config.ts`:

    ```ts
    import { mergeConfig } from 'vitest/config';
    import baseConfig from '@apollogeddon/forgejs/vitest.config.cjs';

    export default mergeConfig(baseConfig, {
      test: {
        // Project-specific overrides
      }
    });
    ```
  </TabItem>

  <TabItem label="Commitlint" icon="seti:git">
    **Commit Message Linting**

    Extend the shared commitlint configuration in `commitlint.config.ts`:

    ```js
    import type { UserConfig } from '@commitlint/types';
    import baseConfig from '@apollogeddon/forgejs/commitlint.config.cjs';

    const Configuration: UserConfig = {
      extends: [baseConfig.extends],
      // Add project-specific rules here
    };

    export default Configuration;
    ```
  </TabItem>

  <TabItem label="Lefthook" icon="laptop">
    **Git Hooks**

    To use the shared Lefthook configuration, copy it into the project root:

    ```bash
    cp node_modules/@apollogeddon/forgejs/lefthook.yml ./lefthook.yml
    ```
  </TabItem>
</Tabs>

## Build & Release

<Tabs>
  <TabItem label="Tsup" icon="node">
    **TypeScript Bundler**

    Import the base configuration and override properties in `tsup.config.ts`:

    ```ts
    import { defineConfig } from "tsup";
    import baseConfig from "@apollogeddon/forgejs/tsup.config.cjs";

    export default defineConfig({
      ...baseConfig,
      // Override or add project-specific properties here
      entry: ["./src/index.ts"],
    });
    ```
  </TabItem>

  <TabItem label="Release" icon="rocket">
    **Release Please**

    Forge.js uses **Release Please** for automated versioning and changelogs. Unlike Semantic Release, it does not require a local `.releaserc.json` file.

    Configuration is handled automatically via the Forge.js reusable workflows.
  </TabItem>

  <TabItem label="Snodeb" icon="linux">
    **Debian Packaging**

    Import the base configuration and override properties in `snodeb.config.ts`:

    ```ts
    import { defineSnodebConfig } from "snodeb";
    import baseConfig from "@apollogeddon/forgejs/snodeb.config.cjs";

    export default defineSnodebConfig({
      ...baseConfig,
      // Override or add project-specific properties here
      files: {
        ...baseConfig.files,
        include: ["dist/index.js"], // Example override
      },
    });
    ```
  </TabItem>

  <TabItem label="Starlight" icon="document">
    **Documentation Site**

    Starlight configuration is scaffolded by the `init` command into `astro.config.mjs`.

    Unlike other tools, Astro configuration is not inherited. You can modify `astro.config.mjs` directly to customise your documentation site.

    Requires: `astro`, `@astrojs/starlight`, `starlight-typedoc`, `sharp`.
  </TabItem>
</Tabs>
